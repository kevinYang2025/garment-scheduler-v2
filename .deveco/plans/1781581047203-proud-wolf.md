## 🔴 严重问题（会导致崩溃、数据错误或功能完全失效）

### S1. SewingPlanDetail.vue:89 — `numFilters` 未定义，运行时 ReferenceError 崩溃

**代码：**
```js
return !!textFilters.value[field]?.applied || !!numFilters.value[field]?.applied
```

**问题：** `numFilters` 从未在文件中声明。每次 `isFilterActive` 被调用时（对所有非日期列），都会抛出 `ReferenceError: numFilters is not defined`。

**后果：** 过滤器激活状态指示器崩溃，可能导致整个组件无法渲染。

**修复：**
```js
// 在文件顶部添加：
const numFilters = ref({})

function onNumFilter(field, f) {
  numFilters.value = { ...numFilters.value, [field]: { ...f, applied: true } }
}
```

### S2. SewingPlanDetail.vue:529,550 + Styles.vue:170-173 — NumberFilter 数据类型不匹配，过滤永远无法命中

**代码：**
```js
// NumberFilter 的 checked Set 存储字符串 "500"
// 但 r[field] 返回数字 500
if (hasVal && !f.checked.has(val)) return false  // Set.has(500) → false（Set里是"500"）
```

**后果：** 用户应用 NumberFilter 后，所有行都被过滤掉，表格显示为空。数字列的过滤功能完全不可用。

**修复：**
```js
const val = r[field] != null ? String(r[field]) : ''
// 或者
if (hasVal && !f.checked.has(String(val))) return false
```

### S6. Styles.vue:181-184 + StyleColorSize.vue:168-183 + FabricLoadingList.vue:199-202 + ExcelTable.vue:104-107 — O(n²log n) 排序，大数据集下浏览器冻结

**代码：**
```js
// 在 sort 比较器内部调用 .filter()，每次比较遍历整个列表
const ac = list.filter(x => x[field] === a[field]).length
const bc = list.filter(x => x[field] === b[field]).length
return (ac - bc) * d
```

**后果：** 1000 条记录时约 2000 万次迭代，5000 条记录时浏览器直接冻结。按计数排序功能在大数据集下不可用。

**修复（通用方案）：** 在排序前预计算频率 Map：
```js
const countMap = new Map()
for (const x of list) countMap.set(x[field], (countMap.get(x[field]) || 0) + 1)
list = [...list].sort((a, b) => ((countMap.get(a[field]) || 0) - (countMap.get(b[field]) || 0)) * d)
```

### S7. WorkCalendar.vue:218-227 — 直接修改响应式对象，失败时 UI 与服务器状态不一致

**代码：**
```js
// @click 内联修改 selectedCalendar.work_days 后直接调 API
selectedCalendar.work_days = newWorkDays
api.updateWorkCalendar(selectedCalendar.id, selectedCalendar)  // 无 .catch()
```

**后果：** (1) API 失败时 UI 已修改但服务器未更新；(2) 无 `.catch()` 导致未处理的 Promise 拒绝。

**修复：** 先调 API，成功后再更新本地状态，并加 try/catch。

### S8. SchedulingStrategy.vue:28-36 — 策略激活循环中 JSON.parse 可抛异常，部分成功导致数据不一致

**代码：**
```js
for (const s of strategies.value) {
  await api.updateStrategy(s.id, { ...s, config: JSON.parse(s.config), active: s.id === id })
  // JSON.parse(s.config) 若 config 不是合法 JSON，整个函数崩溃
  // 前面 N-1 条已更新，后面的未更新 → 状态不一致
}
```

**后果：** 数据损坏：部分策略已切换状态，部分未切换，且本地状态未刷新。

**修复：** (1) 包裹 `JSON.parse` 在 try/catch 中；(2) 应由后端提供原子性"激活策略"接口。

### S9. DispatchReport.vue:314-328 — Blob 套 Blob 导致导出文件损坏

**代码：**

const { data } = await api.exportDispatchReport(params)  // data 已经是 Blob

### S10. MainPlan.vue:254 — `new Date("YYYY-MM-DD")` 解析为 UTC，时区偏移导致日期计算错误

**代码：**
```js
const due = new Date(form.value.due_date)  // "2026-06-20" → UTC 午夜
// UTC-5 时区用户看到 6月19日
```

**后果：** 日期倒推计算在 UTC-X 时区下全部偏移一天，排程日期错误。

**修复：**
```js
function parseLocalDate(str) {
  const [y, m, d] = str.split('-').map(Number)
  return new Date(y, m - 1, d)
}
const due = parseLocalDate(form.value.due_date)
```
