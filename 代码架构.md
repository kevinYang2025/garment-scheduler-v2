# 制衣厂生产计划系统 精简版项目文档
**适配场景**：单人全栈开发、3个核心业务模块、10~20人内部使用
**架构原则**：保留模块化核心骨架，砍掉所有非必要层级与配置，先跑通核心业务，后续再逐步扩展

### 精简说明
1. 砍掉了所有「为了规范而规范」的细分目录，比如前端API不再套子目录、后端实体统一存放
2. 去掉了初期非必需能力：Redis、定时任务、数据库版本迁移、精细化操作日志
3. 保留模块化底线：每个业务模块独立封装，新增功能不侵入老代码，后续扩展不返工
4. 部署配置扁平化，去掉多余目录层级，拿到手就能用

## 第二部分 精简版产品需求文档
### 一、项目概述
#### 1. 项目目标
打造轻量化生产执行系统，覆盖「计划排产 → 多工序报工 → 裁片中转仓 → 缝制生产」全链路，替代手工Excel，实现生产数据实时同步、计划与实际自动对比、进度透明可查。

#### 2. 项目范围
- 包含：用户权限、基础资料、预排总计划、前置加工计划、缝制排程、5类工序报工、裁片仓库、生产看板
- 不含：自动排程算法、采购财务、成品仓、品质检验详情

### 二、用户角色与权限
共7类角色，系统管理员为最高权限账号：

| 角色 | 人数 | 核心权限 |
|------|------|----------|
| 系统管理员 | 1 | 全模块所有权限，管理账号、角色、系统配置，不受任何数据限制 |
| 计划主管 | 1 | 所有生产业务全权限，审核计划、调整排程、查看全厂数据 |
| 计划员 | 3 | 编制总计划、生成前置加工计划、手动排缝制产线 |
| 报工人员 | 6 | 仅能录入、修改本人负责工序的报工数据 |
| 裁片仓管员 | 2 | 裁片出入库确认、库存查询、台账查看 |
| 缝制车间主任 | 5 | 仅查看、修改本车间的缝制计划与报工数据 |

**数据权限规则**：
- 系统管理员：查看所有数据，无限制
- 车间主任：只能看到自己所属车间的数据
- 报工员：只能看到自己负责工序的入口和数据
- 所有操作记录操作人、操作时间，可追溯

### 三、核心业务总流程
```
1. 维护基础资料：款式、面料、车间产线、产能
2. 编制预排总计划，确认后自动生成前置加工计划
3. 手动排布5个缝制车间的生产计划
4. 生产执行：
   裁剪一检报工 → 无后续工序则待入库
               → 有印花/刺绣则生产后二检报工，再待入库
   模板生产 → 模板报工 → 待入库
5. 仓管员确认 → 裁片正式入库
6. 按缝制计划领料出库
   → 需烫标则先烫标报工，再进缝制车间
   → 无需烫标直接进缝制车间
7. 缝制车间报工，自动更新计划进度、计算差异
```

### 四、功能模块说明
#### 1. 系统管理
- **用户管理**：新增编辑用户，绑定角色、所属车间、负责工序，启停账号、重置密码
- **角色管理**：创建角色，分配菜单权限，控制不同角色能看到的页面
- **系统配置**：维护工厂基础参数、业务规则（如是否允许超计划报工）

#### 2. 基础资料
- **款式档案**：维护款号、名称、客户、订单数量、出货日期；配置该款式需经过的工序（印花/刺绣/模板/烫标）；维护各工序标准产能
- **面料管理**：面料档案维护，面料到货记录登记，关联对应款式
- **车间产线**：维护5个缝制车间、各前置工序产线的基础信息与日产能

#### 3. 计划管理
- **预排总计划**：和原来的计划排程方案一致
- **前置加工计划跟进表**：总计划确认后自动生成裁剪、印花、刺绣、模板、烫标的明细计划；展示计划数量、实际完成数、完成率、差异；报工数据自动回写
- **缝制产线排程**：在甘特图手动将款式分配到5个缝制车间，设置生产数量与计划时间；支持表格视图查看，超期计划标红预警

#### 4. 报工管理
通用规则：所有报工关联对应计划，支持当日多次提交，累计计算完成量，提交后自动更新对应计划进度。

1. **裁剪一检报工**：录入合格数、不良数；若款式无后续前置工序，自动生成裁片待入库单
2. **裁剪二检报工**：录入印花/刺绣完成的合格数、不良数；报工后自动生成裁片待入库单
3. **模板报工**：录入模板完成数量，报工后自动生成待入库单
4. **烫标报工**：关联裁片出库单，录入烫标完成数量
5. **缝制车间报工**：录入当日完成数、在制数、不良数；车间主任仅能操作本车间数据
6. **差异计算**：差异数 = 计划数 - 实际完成数，超计划时标黄提示

#### 5. 裁片仓库管理
- **裁片入库**：只能从报工生成的待入库单确认入库，不能手工新增；确认后库存自动增加
- **裁片出库**：关联缝制计划生成领料单；标注是否需要烫标；确认出库后库存自动扣减
- **库存查询**：按款式查看当前库存数量、可用量、待出库量
- **库存台账**：流水展示所有出入库记录，支持按款式、时间筛选

#### 6. 生产看板
- 全厂当日产量、各工序整体达成率
- 5个缝制车间进度排名、当日产量
- 临近出货期的款式预警
- 裁片库库存概览

### 五、非功能要求
1. 页面加载≤2秒，报工提交响应≤1秒，支持20人同时在线
2. 支持Docker一键部署，适配香港服务器，数据库每日自动备份
3. 全接口登录鉴权，HTTPS加密传输，密码加密存储
4. 报工页面操作简化，适合车间人员使用
5. 模块化结构，后续可扩展品质、采购、财务等模块

### 六、起步阶段可延后实现的功能
以下功能第一版可以不做，系统跑通稳定后再加即可：
- 按钮级精细化权限（先做到菜单级）
- 完整操作日志审计
- Redis缓存、定时任务
- 甘特图排程（先只用表格视图）
- 库存盘点、盘盈盘亏
- 批量导入导出Excel
