# 报工管理全面升级计划

## Context

当前报工模块（DispatchReport.vue）仅是一个只读汇总页面，无法录入、编辑、删除报工数据。ActualEntryDialog.vue 虽有录入功能但仅从排程甘特图触发，缺少独立入口。操作者为柬埔寨工人，界面必须极简。粒度到款号级别。

## 目标

将报工管理从"只读汇总页"升级为完整的报工管理模块，覆盖：录入、编辑、删除、批量导入、汇总报表、班组趋势图、计划vs实际对比、导出、进度预警。

---

## Phase 1: 后端 API 补全

**文件**: `garment-server/server.js`

### 1.1 补全 CRUD API

当前只有 GET 和 POST，需补充：

- `PUT /api/actual/:id` — 编辑报工记录
- `DELETE /api/actual/:id` — 删除报工记录（同步清理 schedule_daily ACTUAL 行）
- `GET /api/actual/:id` — 获取单条记录（编辑回填用）

### 1.2 批量导入 API

- `POST /api/actual/batch` — 接收数组批量写入 actual_production，逐条 syncActualToDaily + recalcTaskStatus

### 1.3 汇总增强

增强 `GET /api/dispatch-summary`：
- 支持 `group_by` 参数：`date`（按日）、`style`（按款号）、`workshop`（按车间）、`line_team`（按班组）
- 新增 `GET /api/dispatch-daily-trend` — 按日聚合完成量趋势（用于折线图）
- 新增 `GET /api/dispatch-plan-vs-actual` — 计划 vs 实际对比数据（关联 schedule_daily PLAN vs ACTUAL）

### 1.4 进度预警 API

- `GET /api/dispatch-alerts` — 查询落后于计划的款号（actual < plan 的 schedule_master）

### 1.5 导出 API

- `GET /api/dispatch-export` — 导出 Excel 报表（使用 ExcelJS，项目已依赖）

## Phase 2: 前端报工管理页重构

**文件**: `garment-web/src/views/DispatchReport.vue`（重写）

### 2.1 布局结构

```
┌─────────────────────────────────────────┐
│ 报工管理                    [录入] [导入] [导出] │
├─────────────────────────────────────────┤
│ 汇总卡片：总完成量 | 次品 | 合格率 | 滞后款数  │
├─────────────────────────────────────────┤

│ [录入列表 Tab]                            │

│ 表格：日期|款号|完成量|次品|合格率|操作(编辑/删除)│
│ 分页                                      │
├─────────────────────────────────────────┤
│ [趋势分析 Tab]                            │

```

### 2.2 极简录入对话框（重写 ActualEntryDialog.vue）

针对柬埔寨工人操作，需极简：
- **只有必填项**：款号（下拉选择）、完成数量
- **自动填充**：日期默认今天、车间/班组从上次录入记忆
- **次品可选**：默认0，不强制填
- **大按钮、大字号**：适合触屏/快速操作
- **录入后不清空款号**：方便同一款号重复录入

### 2.3 批量导入对话框

新建 `garment-web/src/components/ImportActualDialog.vue`：
- 支持 Excel 模板下载
- 拖拽上传 Excel
- 预览确认后批量写入
- 错误行高亮提示

### 2.4 编辑对话框

复用录入对话框，增加编辑模式：
- 打开时回填已有数据
- PUT 提交更新

### 2.5 导出功能

- 点击导出按钮 → 后端生成 Excel → 前端下载
- 导出内容与当前筛选条件一致

## 验证方式

1. 启动后端 `cd garment-server && node server.js`
2. 启动前端 `cd garment-web && npm run dev`
3. 测试流程：
   - 打开报工管理页 → 点击"录入" → 选择款号 → 输入完成数量 → 提交
   - 编辑刚录入的记录 → 修改数量 → 保存
   - 删除一条记录 → 确认 schedule_daily ACTUAL 行也清除
   - Excel 导入多条记录 → 验证写入正确
   - 切换"趋势分析"Tab → 查看折线图
   - 切换"计划vs实际"Tab → 查看柱状图
   - 点击"导出" → 下载 Excel → 验证内容
   - 确认滞后款号在预警卡片和表格中标红
