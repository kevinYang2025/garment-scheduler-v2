# 印花排程表格重构计划

## 需求
将「计划管理/二次加工/印花排程」的排程表格，替换为与「缝制排程/班组缝制计划」(SewingPlanDetail.vue) **完全相同格式**的表格。

**行粒度**：每个款式的 每个颜色+每个尺码 = 一行（三行子行：计划/实际/差异）
**数据来源**：
- styles 表 → printing='是' 的款式 (style_no, product_name, printing_daily_output)
- style_color_size 表 → 按 style_no 查 color + size_spec + plan_qty
- main_plan 表 → printing_start + printing_end
**创建方式**：自动生成（主计划排产时）+ 手动新建

## Step 1: 后端 — 新增 API 端点

### 1.1 `GET /api/printing-plan-data` (server.js)

聚合三表数据，返回每个 color+size 一行的列表：

```js
app.get('/api/printing-plan-data', (req, res) => {
  // 1. styles WHERE printing='是' → styleMap[style_no] = { id, style_no, product_name, printing_daily_output, due_date }
  // 2. main_plan WHERE printing_start != '' → planMap[style_no] = { printing_start, printing_end }
  // 3. style_color_size ORDER BY style_no, color, size_spec
  // 4. 对每条 color_size 记录：
  //    - 查 styleMap 是否有该款号（无则跳过）
  //    - 查 planMap 获取日期
  //    - 查 schedule_master 是否已有记录（有则用已有数据，无则返回虚拟记录 _virtual:true）
  // 返回: [{ id, style_no, product_name, color, size_spec, plan_qty, daily_target, plan_start, plan_end, ... }]
})
```

## Step 2: 后端 — 自动排产生成印花记录

**文件**: `garment-server/db.js` → `autoSchedule()` 函数（在事务内）

在现有自动排产逻辑末尾添加：

```js
// 自动为 printing='是' 的款式生成印花排程记录
const printingStyles = db.prepare("SELECT * FROM styles WHERE printing='是' AND printing_daily_output > 0").all()
const mainPlans = db.prepare("SELECT style_no, printing_start, printing_end FROM main_plan WHERE printing_start != ''").all()
const planMap = {}
for (const p of mainPlans) { if (!planMap[p.style_no]) planMap[p.style_no] = p }

for (const style of printingStyles) {
  const mp = planMap[style.style_no]
  if (!mp || !mp.printing_start || !mp.printing_end) continue
  const colorSizes = db.prepare("SELECT * FROM style_color_size WHERE style_no = ?").all(style.style_no)
  for (const cs of colorSizes) {
    const exists = db.get("SELECT id FROM schedule_master WHERE schedule_type='secondary' AND secondary_type='printing' AND style_no=? AND color=? AND size_spec=?", [cs.style_no, cs.color, cs.size_spec])
    if (exists || !cs.plan_qty) continue
    const r = db.run(`INSERT INTO schedule_master (schedule_type, secondary_type, style_id, style_no, product_name, color, size_spec, plan_qty, cutting_plan_qty, plan_start, plan_end, daily_target, due_date) VALUES ('secondary','printing',?,?,?,?,?,?,?,?,?,?,?)`,
      [style.id, style.style_no, style.product_name, cs.color, cs.size_spec, cs.plan_qty, cs.plan_qty, mp.printing_start, mp.printing_end, style.printing_daily_output, style.due_date || ''])
    generatePlanRows(r.lastInsertRowid, mp.printing_start, mp.printing_end, cs.plan_qty)
  }
}
```

## Step 4: 新建 PrintingPlanDetail.vue

**文件**: `garment-web/src/views/PrintingPlanDetail.vue`

从 SewingPlanDetail.vue 复制并做以下适配：

### 4.1 列名映射

| SewingPlanDetail (缝制) | PrintingPlanDetail (印花) |
|------------------------|--------------------------|
| 裁床计划数量 `cutting_plan_qty` | 原单量 `cutting_plan_qty`（同字段，改label） |
| 目标日产量 `daily_target` | 印花日产量 `daily_target`（同字段，改label） |
| 缝制开始 `plan_start` | 印花开始 `plan_start`（同字段，改label） |
| 缝制结束 `plan_end` | 印花结束 `plan_end`（同字段，改label） |
| *(无)* | **颜色** `color`（新增列，在品名后） |
| *(无)* | **尺码** `size_spec`（新增列，在颜色后） |

### 4.5 款式选择器过滤

StylePicker 只显示 `printing='是'` 的款式（可在前端过滤或后端加参数）。

### 4.6 保留的功能

- 三行模型（计划/实际/差异）
- 日期窗口导航（◀ 今天 ▶，±7天偏移）
- 今天列高亮 `#e0d4ff`
- sticky 左侧固定列
- 内联编辑（车间/班组/数量/日期）
- 筛选系统（TextFilter/NumberFilter/DateFilter）
- 拖拽滚动
- 导入导出（改列映射）
- plan_end 自动计算（plan_start + ceil(qty/daily) - 1）
